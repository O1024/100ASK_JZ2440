# 实验五：系统时钟配置与 UART0 通信

本实验将掌握 S3C2440 的时钟系统 (PLL) 与 UART 模块。首先配置 MPLL 将系统主频从 12MHz 提升至 400MHz，在此基础上实现稳定的 115200 波特率串口通信。

---

## 🎯 实验目标
1.  理解 S3C2440 的时钟体系：FCLK、HCLK 和 PCLK。
2.  掌握 **MPLL** 配置与 **CLKDIVN** 分频设置。
3.  掌握 GPIO 复用功能配置（GPHCON）。
4.  掌握在 50MHz PCLK 环境下计算 **115200** 波特率的方法。
5.  通过轮询方式实现字符的双向收发与回显。

---

## 📚 硬件原理剖析

### 1. 时钟架构
*   **Fin (12MHz)**：外部输入晶振。
*   **FCLK (400MHz)**：提供给 CPU 核。
*   **HCLK (100MHz)**：提供给 AHB 总线（存储控制器、中断控制器、LCD 等）。
*   **PCLK (50MHz)**：提供给 APB 总线（UART、Timer、GPIO 等）。

### 2. 串口连接 (UART0)
参考 `docs/schematics/JZ2440_V3电路图.pdf`：
*   **GPH2** -> TXD0（发送）
*   **GPH3** -> RXD0（接收）
*   **转换芯片**：板载 PL2303 用于 USB 转串口。

---

## 💻 软件实现细节

### 1. 时钟初始化流程 (`src/cpu/start.S`)
1.  **LOCKTIME**：设置 PLL 稳定等待时间。
2.  **CLKDIVN**：设置分频比为 1:4:8。
3.  **CPU 异步模式**：通过 CP15 协处理器指令切换 CPU 总线模式（FCLK != HCLK 时必须执行）。
4.  **MPLLCON**：写入 MDIV=92, PDIV=1, SDIV=1，启动升频。

### 2. 波特率计算 (`src/main.c`)
公式：`UBRDIVn = (int)(PCLK / (bps * 16)) - 1`

**本实验参数**：
*   **PCLK = 50MHz**
*   目标 bps = 115200
*   `UBRDIV0 = (50,000,000 / (115200 * 16)) - 1 ≈ 26.13` -> 取 **26**。
*   **优势**：相比 12MHz 时的 8.5% 误差，现在的误差仅为 **0.4%**，通信极度稳定。

### 2. 调试环境
1.  启动 OpenOCD。
2.  使用 GDB 加载 `jz2440.elf`。
3.  在 PC 端打开串口调试助手，参数：**波特率 115200，8N1，无流控**。

---

## ⚠️ 注意事项
*   **异步模式**：在 `start.S` 中若不切换 CPU 到异步总线模式，系统在 MPLL 开启后会立即挂起。
*   **终端显示**：串口回显逻辑中，检测到 `\r` 会补充 `\n`，以适配大多数 PC 端串口助手的换行逻辑。
