# 实验二：按键控制 LED (轮询模式)

本实验基于流水灯实验，进一步学习 S3C2440 的 GPIO 输入功能。通过不断扫描（轮询）按键引脚的状态，实时控制 LED 的亮灭。

---

## 🎯 实验目标
1.  掌握 GPIO 输入模式的寄存器配置。
2.  理解**内部上拉电阻 (Pull-up)** 的原理及在输入电路中的必要性。
3.  学习基础的**按键消抖 (Debounce)** 算法（软件延时法）。
4.  实现多按键组合控制多 LED。

---

## 📚 硬件原理剖析

### 1. 按键引脚连接
参考 `docs/schematics/JZ2440_V3电路图.pdf`：
*   **S2** -> **GPF0**
*   **S3** -> **GPF2**
*   **S4** -> **GPG3**
*   **S5** -> **GPG11**

### 2. 控制逻辑
*   **按键状态**：
    - 未按下：引脚由于上拉电阻保持 **高电平 (1)**。
    - 按下：引脚直接接地，呈现 **低电平 (0)**。
*   **LED 关联**：
    - S2 (GPF0) 控制 LED1 (GPF4)
    - S3 (GPF2) 控制 LED2 (GPF5)
    - S4 (GPG3) 控制 LED4 (GPF6)

---

## 💻 软件实现细节

### 1. 寄存器配置 (`key_init`)
*   **GPFCON/GPGCON**：必须将对应位设置为 `00` (Input 模式)。
*   **内部上拉 (`GPFUP/GPGUP`)**：
    - 向对应位写入 `0` 以开启内部上拉。
    - **原因**：虽然 JZ2440 有外部上拉，但开启内部上拉能进一步增强信号的稳定性，防止引脚浮空导致的电平跳变。

### 2. 软件消抖逻辑
机械按键在闭合瞬间会产生电平抖动。我们在检测到低电平时加入一小段延时，再次确认电平，以滤除误触发：

```c
if (!(GPFDAT & (1 << 0))) { // 检测到 S2 按下
    delay(10000);           // 软件延时消抖
    if (!(GPFDAT & (1 << 0))) { // 再次确认
        GPFDAT &= ~(1 << 4); // 点亮 LED1
    }
}
```

### 💡 知识点：按键消抖的几种方案

由于机械按键在按下或松开的瞬间，接触点会发生弹性碰撞，导致电平信号在短时间内（通常 5ms~20ms）剧烈抖动。为了避免程序误判，通常有以下几种处理方案：

#### 1. 软件阻塞式延时（本实验采用）
*   **逻辑**：检测到变化 -> 延时 20ms -> 再次检测 -> 确认电平。
*   **优点**：实现极其简单，适合裸机简单逻辑。
*   **缺点**：延时期间 CPU 处于忙等状态，无法处理其他任务，效率极低。

#### 2. 软件非阻塞状态机（推荐）
*   **逻辑**：利用硬件定时器产生固定周期（如 10ms）的中断。在中断中读取按键电平，如果连续两次采样电平一致且发生变化，则触发事件。
*   **优点**：完全不占用 CPU 正常运行时间，响应速度快，支持复杂多任务环境。
*   **对应分支**：可参考 [lab/io_interrupt](../../tree/lab/io_interrupt) 中关于异步处理的探索。

#### 3. 硬件滤波消抖
*   **逻辑**：
    - **RC 滤波**：在按键两端并联电容，利用电容充放电的时间常数滤除高频脉冲。
    - **施密特触发器**：利用其滞回特性对毛糙的波形进行整形。
*   **优点**：最可靠，完全不依赖软件代码。
*   **缺点**：增加硬件成本和 PCB 面积。
