# 实验二：按键控制 LED (轮询模式)

本实验基于流水灯实验，进一步学习 S3C2440 的 GPIO 输入功能。通过不断扫描（轮询）按键引脚的状态，实时控制 LED 的亮灭。

---

## 🎯 实验目标
1.  掌握 GPIO 输入模式的寄存器配置。
2.  理解**内部上拉电阻 (Pull-up)** 的原理及在输入电路中的必要性。
3.  学习基础的**按键消抖 (Debounce)** 算法（软件延时法）。
4.  实现多按键组合控制多 LED。

---

## 📚 硬件原理剖析

### 1. 按键引脚连接
参考 `docs/schematics/JZ2440_V3电路图.pdf`：
*   **S2** -> **GPF0**
*   **S3** -> **GPF2**
*   **S4** -> **GPG3**
*   **S5** -> **GPG11**

### 2. 控制逻辑
*   **按键状态**：
    - 未按下：引脚由于上拉电阻保持 **高电平 (1)**。
    - 按下：引脚直接接地，呈现 **低电平 (0)**。
*   **LED 关联**：
    - S2 (GPF0) 控制 LED1 (GPF4)
    - S3 (GPF2) 控制 LED2 (GPF5)
    - S4 (GPG3) 控制 LED4 (GPF6)

---

## 💻 软件实现细节

### 1. 寄存器配置 (`key_init`)
*   **GPFCON/GPGCON**：必须将对应位设置为 `00` (Input 模式)。
*   **内部上拉 (`GPFUP/GPGUP`)**：
    - 向对应位写入 `0` 以开启内部上拉。
    - **原因**：虽然 JZ2440 有外部上拉，但开启内部上拉能进一步增强信号的稳定性，防止引脚浮空导致的电平跳变。

### 2. 软件消抖逻辑
机械按键在闭合瞬间会产生电平抖动。我们在检测到低电平时加入一小段延时，再次确认电平，以滤除误触发：

```c
if (!(GPFDAT & (1 << 0))) { // 检测到 S2 按下
    delay(10000);           // 软件延时消抖
    if (!(GPFDAT & (1 << 0))) { // 再次确认
        GPFDAT &= ~(1 << 4); // 点亮 LED1
    }
}
```

## 🛠️ OpenOCD 配置文件详解 (`scripts/jz2440.cfg`)

本分支沿用了经过稳定性优化的调试配置：
*   **速率限制**：`adapter speed 200`。由于 CPU 运行在 12MHz 晶振下，低速 JTAG 能确保指令加载不丢包。
*   **复位模式**：`reset_config trst_only`。针对 JZ2440 仅引出 TRST 复位线的硬件特征，通过 JTAG 软指令控制内核复位。
*   **下载加速**：配置了 `0x40000000` 作为 SRAM 工作区，并禁用了 DCC 模式以提高 `load` 命令的兼容性。

## 🚀 编译与运行
1.  **编译**：`make clean && make`。
2.  **启动网关**：`openocd -f scripts/jz2440.cfg`。
3.  **下载调试**：`gdb-multiarch jz2440.elf -x scripts/gdbinit`。
