# 实验一：SRAM 调试环境搭建与流水灯实现

本实验是 JZ2440 裸机开发的第一步。目标是搭建完整的交叉编译与硬件调试环境，并通过控制 GPIO 实现最基础的 LED 流水灯逻辑。

---

## 🎯 实验目标
1.  搭建 `arm-none-eabi-gcc` 交叉编译环境。
2.  配置 OpenOCD 与 GDB，实现程序通过 JTAG 直接下载到 SoC 内部 SRAM 运行。
3.  掌握 S3C2440 GPIO 的寄存器配置方法（GPFCON/GPFDAT）。
4.  理解流水灯的硬件电路原理。

---

## 🛠️ 准备工作

### 1. 硬件需求
*   **开发板**：JZ2440 V3 (S3C2440A)。
*   **调试器**：OpenJTAG (或其它兼容 FT2232 的 JTAG 调试器)。
*   **连接**：使用 JTAG 排线连接调试器与板子的 JTAG 接口，并将板子设为 **NOR 启动**（仅为确保复位后 CPU 处于可控状态）。

### 2. 软件环境
*   **工具链**：`arm-none-eabi-gcc` (建议 13.2.1 或更高)。
*   **调试器**：`openocd` 与 `gdb-multiarch`。
*   **驱动**：确保系统识别 JTAG 调试器的 FTDI 芯片。

---

## 📚 硬件原理剖析

### 1. LED 接口连接
参考 `docs/schematics/JZ2440_V3电路图.pdf` 第 3 页，LED 连接在 **PORT F**：
*   **LED1** -> **GPF4**
*   **LED2** -> **GPF5**
*   **LED4** -> **GPF6**

### 2. GPIO 寄存器细节
控制 LED 需要操作两个核心寄存器：
*   **GPFCON (0x56000050)**：配置引脚功能。
    *   每 2 位控制一个引脚。对于 GPF4，对应位为 `[9:8]`。
    *   `00` = 输入，`01` = 输出，`10` = 中断功能。
    *   本实验需设置 `[9:8]`, `[11:10]`, `[13:12]` 为 `01`。
*   **GPFDAT (0x56000054)**：控制引脚电平。
    *   对应位写 `0` 输出低电平，点亮 LED。
    *   对应位写 `1` 输出高电平，熄灭 LED。

## 💻 软件实现细节

### 1. 启动代码 (`src/cpu/start.S`)
当开发板复位或通过调试器 Halt 时，CPU 首先执行此段汇编：
1.  **关闭看门狗**：向 `WTCON (0x53000000)` 写入 `0`。若不关闭，系统会在约 5 秒后由于计时器溢出而自动重启，导致调试中断。
2.  **设置栈指针 (SP)**：`ldr sp, =4096`。C 语言的函数调用和局部变量依赖栈空间。S3C2440 启动时内部 SRAM（Steppingstone）映射在 `0x0` 地址，大小为 4KB，故将栈顶设在 4096 处。
3.  **跳转到 C 入口**：`bl main`。

### 2. 延时函数说明
由于此时系统运行在外部晶振频率（12MHz），且尚未开启 Cache，指令执行速度较慢。`delay(100000)` 大约产生几百毫秒的肉眼可见延迟。

## 🚀 编译、烧录与调试

### 步骤 1：编译
执行 `make` 会经历以下流程：
1.  **汇编/编译**：将 `.S` 和 `.c` 转换为目标文件 `.o`。
2.  **链接**：使用 `-Ttext 0` 将程序链接到地址 `0x0`。
3.  **格式转换**：`objcopy` 将 ELF 文件提取为纯二进制 `.bin`（用于烧录）。

### 步骤 2：启动 OpenOCD
```bash
openocd -f scripts/jz2440.cfg
```
该命令会建立 GDB 与物理硬件之间的桥梁（默认端口 3333）。

### 步骤 3：加载运行 (GDB)
```bash
gdb-multiarch jz2440.elf -x scripts/gdbinit
```
**`scripts/gdbinit` 做了什么？**
*   `target remote localhost:3333`：连接调试网关。
*   `monitor halt`：强制 CPU 停止运行。
*   `load`：将二进制代码通过 JTAG 写入 SoC 内部 SRAM。
*   `break _start`：在汇编入口设置断点。

输入 `c` (continue) 后，LED 将按照代码逻辑循环点亮。

## ⚠️ 常见问题排查 (FAQ)
*   **Q: OpenOCD 报错 "JTAG tap: s3c2440.cpu tap/device found: 0x00000000"?**
    *   A: 检查排线是否插紧，确保开发板已上电。尝试在执行命令的同时按住板子的复位键。
*   **Q: 程序可以 load 成功但 LED 不亮？**
    *   A: 检查 `s3c2440_soc.h` 中的寄存器地址是否正确。确保板子处于 NOR 启动模式。
*   **Q: 为什么 load 地址是 0x0?**
    *   A: 因为在 SRAM 调试模式下，S3C2440 的 Steppingstone 区域映射在地址空间的最前端。
