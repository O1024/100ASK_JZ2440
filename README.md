# 实验三：IO 中断控制 (EINT) 与消抖处理

本实验将流水灯和按键实验升级为**中断驱动模式**。通过学习 ARM 异常处理机制，实现更高效的按键响应，并深入探讨机械按键在中断模式下的消抖策略。

---

## 🎯 实验目标
1.  理解 ARM920T 的**异常向量表**（Exception Vector Table）布局。
2.  掌握 S3C2440 **中断控制器**（SRCPND/INTMSK/INTPND）的配置流程。
3.  学习**外部中断 (EINT)** 的触发配置与二级挂起处理。
4.  掌握中断服务程序 (ISR) 中的现场保护与“写 1 清零”规范。
5.  实现稳定可靠的按键中断消抖逻辑。

---

## 📚 硬件原理剖析

### 1. 中断引脚分配
参考 `docs/schematics/JZ2440_V3电路图.pdf`：
*   **S2** -> GPF0 (**EINT0**)
*   **S3** -> GPF2 (**EINT2**)
*   **S4** -> GPG3 (**EINT11**)
*   **S5** -> GPG11 (**EINT19**)

### 2. S3C2440 中断结构
*   **一级中断**：EINT0, EINT2 直接对应中断控制器的 bit 0 和 bit 2。
*   **二级中断**：EINT11, EINT19 汇聚到 `EINT8_23` (bit 5)，需要通过 `EINTPND` 进一步区分。

---

## 💻 软件实现细节

### 1. 异常向量表 (`src/cpu/start.S`)
当硬件产生 IRQ 中断时，CPU 会强制跳转到 `0x18` 地址。我们需要在此放置跳转指令：
```armasm
_start:
    b reset          /* 0x00 */
    ...
    ldr pc, =irq_handler /* 0x18: IRQ 入口 */
```

### 2. 中断初始化 (`key_eint_init`)
1.  **引脚功能**：配置 `GPFCON/GPGCON` 为中断模式 (`10`)。
2.  **触发方式**：配置 `EXTINTn` 寄存器为**下降沿触发**。
3.  **屏蔽位**：
    - 开启二级屏蔽：`EINTMASK` 允许 11 和 19 位。
    - 开启一级屏蔽：`INTMSK` 允许 0, 2, 5 位。

### 3. 消抖与清理逻辑
本实验采用**同步阻塞消抖**：
1.  **进入 ISR**：首先执行一段延时（如 20ms）。
2.  **状态校验**：延时后检查 `GPFDAT/GPGDAT`，确认按键仍按下。
3.  **标志位清理 (核心)**：遵循“先清二级源，再清一级源”的原则，向寄存器对应位**写入 1**。
