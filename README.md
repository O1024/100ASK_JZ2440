# 实验四：DMA 内存拷贝 (Memcpy)

本实验通过 S3C2440 的 DMA 控制器实现 SDRAM 之间的高速数据传输，并对比 CPU 软件拷贝与 DMA 硬件拷贝的性能。

---

## 🎯 实验目标
1.  **实现高速拷贝**：使用 DMA Ch3 在 SDRAM 空间内搬运 1MB 数据。
2.  **性能对比**：通过 LED1 (CPU) 和 LED2 (DMA) 的亮起时长直观对比拷贝效率。
3.  **串口调试**：通过 UART 实时监控传输状态和数据校验结果。

---

## 📚 关键技术点：DCON 寄存器陷阱
在 S3C2440 中，配置 **Software Request** 模式时，`DCON` 寄存器的位定义极易被误解：

| 位 (Bit) | 属性 | 设置值 | 说明 |
| :--- | :--- | :--- | :--- |
| **[31]** | Handshake | 1 | 虽然是 SW 模式，但在 100MHz HCLK 下 Handshake 模式更稳定。 |
| **[27]** | Whole Service | 1 | 必须开启，以确保一次触发完成所有传输。 |
| **[23]** | **SWHW_SEL** | **0** | **关键陷阱**：0 代表 Software Request，1 代表 Hardware。 |
| **[22]** | **RELOAD** | **1** | **关键陷阱**：0 代表开启自动重载，1 代表关闭（单次拷贝需关闭）。 |

---

## 💻 软件实现流程
1.  **初始化**：开启 DMA 模块时钟 (`CLKCON[14]`)，初始化 UART。
2.  **源/目的设置**：`DISRC3` = 0x30100000, `DIDST3` = 0x30200000。
3.  **触发序列**：
    *   写入 `DCON3`。
    *   写入 `DMASKTRIG3 = 3` (同时设置 ON 位和 SW_TRIG 位)。
4.  **状态轮询**：循环检查 `DSTAT3` 的 `CURR_TC` (位 [19:0])，直到其归零。

---

## 🚀 验证方法
1.  **肉眼观察**：LED1 (左) 亮起时间较长，LED2 (中) 亮起时间极短，证明 DMA 效率远高于 CPU 循环。
2.  **串口输出**：观察 `SUCCESS: DMA is working!` 字样及校验结果。

---

## 📝 实验备注
*   **硬件平台**：JZ2440 V3 (S3C2440A)。
*   **SDRAM 时序**：确保 `BANKCON6` 设为 CL=3 (0x18011)，且代码已正确重定位。
